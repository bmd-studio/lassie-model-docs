Class Account_model

The Account Model handles the money accounts that are linked with persons or organisations. There are currently three types of accounts: first account (person), second account (person) and a general account (organisation).

Verify whether an account is allowed to make a transaction Set the preprocessed flag to false when the transaction is not in the database before checking this The Transaction Library works this way: all the transactions are added and after that the balance is checked if the balance appears to be invalid all these transactions are deleted again.

However, the refunding of tokens doesn't work this way. A transaction is not made before and therefore the delta balance should be added to the current balance by setting the is_preprocessed flag to FALSE
